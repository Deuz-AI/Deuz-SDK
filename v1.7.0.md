# v1.7.0 — "Hafızalı, Hesaplı, Kopmaz Chatbot"

> **Kapsam kararı (2026-07-16):** "Çekirdek + differentiator" kesimi — parite çekirdeği (6 hamle) + Vercel'de bile olmayan 6 differentiator + 1 stretch. ~22 kişi-hafta. Vue/Svelte, OpenAI Responses background-resume ve smoothStream **1.7.1'e ertelendi**. Üst bağlam: `p.md` (2026-07-14 benchmark: Deuz 15 SDK içinde 13., chatbot senaryosunda 71 puanla 7.).
>
> **Kanıt disiplini:** Aşağıdaki her "Vercel'de yok" iddiası 2026-07-16'da AI SDK 7 resmî dokümanlarına karşı kuşkucu ajanlarla doğrulandı; çürüyen iddialar (runtime data-part doğrulaması, çok-istemci canlı izleme) listeden çıkarıldı veya parite maddesine indirildi.

---

## Hedef

Chatbot senaryosunda **71 → ~78–79** (senaryo sıralamasında 7. → **4.**), topluluk-dışı ağırlıklı puanda 78,4 → **87,4** (Vercel 91,1 ile fark 12,7'den 3,7'ye iner; kalan fark 1.7.1 pariteleriyle kapanır). Pazarlama omurgası: **"Vercel AI SDK'da bile olmayan 6 şey"** — hepsi kanıt linkli.

Konumlandırma: Vercel'in chatbot yüzeyine yetişmeye çalışan bir kopya değil; **hafıza + maliyet + dayanıklılık** üçlüsünü kütüphane içinde, vendor'suz veren tek SDK.

## Görev Listesi

### Parite çekirdeği (~14,5 kişi-hafta)

- [x] **P1 — Resumable UI wire v2 + `StreamStateStore` seam** *(3 pw)*
  Wire v2'de monoton sequence id; broadcaster pump'ı canonical StreamPart'ları 2-metodlu `StreamStateStore(append/read)` seam'ine yazar (SessionStore deseni, `deps` üzerinden). `toDeuzStreamResponse` Last-Event-ID'den replay; `readDeuzStream`'e backoff'lu otomatik reconnect; **çoklu istemci aynı stream'i canlı izleyebilir** (Vercel paritesi). In-memory default + Redis/Supabase örnek adaptörleri. Kabul: golden-replay kopma-noktası testleri; edge-safe; zero-dep.
- [x] **P2 — `ChatStore` persistence** *(2 pw)*
  `loadChat`/`saveChat` seam'i (chatId + zorunlu scope, memory.ts scope modeliyle tutarlı); loop/useChat `onFinish`'te otomatik persist (immutable history → diff-append). Node yüzeyinde SQLite/JSONL referansı; docs'ta Supabase adaptörü. `useChat({ chatId, initialMessages })`.
- [x] **P3 — Typed data parts + tool durum makinesi + citation (wire v2)** *(2,5 pw)*
  `data-{name}` part tipi + sunucuda `writeData(name, payload)`; şema Standard Schema ile (schema/bridge.ts mevcut) — **akış sırasında otomatik doğrulama opsiyonu** (Vercel'de validateUIMessages manuel). Tool part'larına `ToolStateDelta` durum makinesi (types/stream.ts minor, kontrat ratchet). RAG citation built-in data part.
- [ ] **P4 — `create-deuz-chat` scaffold + headless React bileşenleri** *(4 pw)*
  Ayrı paket (çekirdek zero-dep kalır): unstyled MessageList, PromptInput, ToolApprovalCard, StreamingIndicator, **CostBadge** (D2'ye bağlı). `npx create-deuz-chat`: Next.js + Supabase şablonu, ChatStore + resume kablolu.
- [ ] **P5 — Docs sitesini yayına al + ücretsiz-modelli canlı playground** *(2 pw)*
  Statik yayın; her özellik için tek-tık StackBlitz örneği; playground default'u ücretsiz model (GLM Flash / Gemini free-tier); bench/ sonuçları ana sayfada.
- [x] **P6 — `useChat`'e regenerate + edit&branch** *(1 pw)*
  `regenerate(messageId?)`, `editAndResend(messageId, text)`; immutable history → branch = prefix'ten yeni dizi; ChatStore'a opsiyonel parentId.

### Differentiator'lar (~7,5 kişi-hafta)

- [x] **D1 — `useChat({ memory })` yerleşik hafıza** *(2 pw)*
  Memory pipeline'ı (mevcut memory.ts) chat loop'una bağla: tur sonunda extract→reconcile (async, koşuyu bloklamaz), istek öncesi recall → system-context enjeksiyonu; scope = chatId/userId zorunlu. Kabul: iki oturum arası hatırlama E2E testi; kapalıyken sıfır ek maliyet.
- [x] **D2 — Canlı USD maliyet part'ı** *(1 pw)*
  onUsage + priceProvider'dan kümülatif `cost` data part'ı (wire v2); cache-hit tasarrufu ayrı alan. useChat'te `cost` state'i; CostBadge bileşeni.
- [x] **D3 — Sohbet bütçe korkuluğu** *(1 pw)*
  `budget: { usd?, tokens? }` çağrı opsiyonu → mevcut stop condition'lara bağlanır; aşımda tipli `budget-exceeded` part + `providerMetadata.deuz.stoppedBy`; UI'da devam-onayı deseni (docs).
- [x] **D4 — HMAC-imzalı onay uçtan uca chat UI** *(0,5 pw)*
  ToolApprovalCard `createApprovalSigner` ile imzalı verdict üretir; sunucu resume'da doğrular; docs'ta güven-sınırı rehberi. (Altyapı 1.5'ten beri hazır — bu iş UI kablolaması + rehber.)
- [x] **D5 — Durable × resumable entegrasyonu** *(1 pw)*
  `session` + `StreamStateStore` birlikte: yenileme ortasında koşu checkpoint'ten sürer, istemci kaldığı seq'ten bağlanır. Kabul: "F5 ortasında tool loop" E2E golden testi; docs'ta tek sayfa "kopmaz chatbot" rehberi.
- [x] **D6 — `fallbackModels` cross-provider failover** *(2 pw)*
  Middleware (`withFallback([model1, model2])`) + `fallbackModels` çağrı opsiyonu: ilk-bayt-öncesi hata/breaker açıkken sıradaki modele kanonik geçmişle geç; `providerMetadata.deuz.failedOver` işareti; per-model breaker (mevcut resilience) ile entegre. Kabul: sağlayıcı-A 529 → sağlayıcı-B tamamlar senaryosu deterministik testte.

### Stretch (kapasite kalırsa, ~2 pw)

- [ ] **S1 — `createQwen`/`createGLM` factory'leri + cache-hit-farkındalıklı fiyat kataloğu**
  chat_completions wire üstüne iki ince factory (config-symbol deseni); registry quirk'leri: GLM `tool_choice:'auto'` kısıtı, Qwen 1M context; pricing'e cache-hit/miss fiyatları. Dar iddia: "tipli quirk garantisi + cache-farkındalıklı maliyet kimsede yok."

## Uygulama disiplini

- Yeni subpath'ler (ui v2 genişlemesi, olası `chat` yardımcıları) **6-yer lockstep** prosedürüne uyar (exports + tsup + edge + api-contract regen + eslint node listeleri + runtime-compat regex — bkz. CLAUDE.md).
- Çekirdek edge-safe kalır: StreamStateStore/ChatStore/memory bağlantıları `Dependencies` seam'i üzerinden; `Date.now`/`Math.random` yasağı geçerli (pacing/backoff `deps.clock`).
- `streamChat` senkron-döner/asla-throw-etmez (G2) ve key önceliği (G1) değişmez; wire v2 **additive** — v1 istemcileri çalışmaya devam eder (version negotiation).
- Her hamle golden-replay testli; D5/P1 için kopma-noktası fixture'ları; bütçe/maliyet yolunda `maskSecret`/redaction P0 invariant'ları korunur.
