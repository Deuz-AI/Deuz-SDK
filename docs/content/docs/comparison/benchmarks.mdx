This page compares `@deuz-sdk/core` against the four TypeScript AI SDKs / agent frameworks you are most likely to shortlist in 2026. Every claim is either **code-level verified** (we scanned the codebase and ran the checks) or **docs-level verified** (official documentation, version-pinned, researched 2026-07-07) — each section says which. The scoring rubric is ours and the raw sheets live in this repository, so every cell can be re-derived or disputed.

| **Tier 2 — docs-level** (official docs + release pages, versions pinned, 2026-07-07) | Mastra `@mastra/core@1.46.0` · LangGraph.js `@langchain/langgraph` · OpenAI Agents SDK `@openai/agents@0.12.1` |

## TL;DR — what each one is

| SDK | In one sentence | Reach for it when |
| --- | --- | --- |
| **`@deuz-sdk/core`** | Zero-dependency, edge-safe SDK primitives with one canonical stream, a free-function agent loop, and deterministic-by-design testing | You want to own the wire, the loop and the cost accounting; edge/CLI targets; replayable tests |
| **Vercel AI SDK 7** | The broadest ecosystem: primitives plus agent classes, ~48 providers, 5+ UI frameworks | You want maximum provider/framework coverage and Vercel-native durable workflows |
| **Mastra** | Batteries-included TypeScript agent *framework* — workflows, memory, RAG, Studio in one package | A TS team wants a Rails-like, all-in-one agent backend with managed structure |
| **LangGraph.js** | A low-level graph runtime — explicit nodes/edges with checkpoints at every super-step | Enterprise workflows needing durable graphs, audit trails and time-travel debugging |
| **OpenAI Agents SDK (JS)** | A lightweight runner: agents, handoffs, guardrails, sessions, tracing | You are committed to OpenAI's stack and want the vendor's own minimal orchestration |

These tools sit at different abstraction levels — Deuz and AI SDK are SDKs, Mastra is a full framework, LangGraph is a runtime, the Agents SDK is a thin runner — so a single ranked score would be misleading. The matrix below maps trade-offs; measured scores exist only for the pair we code-verified.

## Profiles — what we verified about each

### Vercel AI SDK 7 — Tier 1, code-level

`ai@7.0.14` (GA 2026-06-25), verified 2026-07-04. Ahead of Deuz today on: durable `WorkflowAgent` (tied to the Vercel Workflow DevKit runtime — `'use workflow'` directives, serializable context), `@ai-sdk/otel`, `@ai-sdk/tui` terminal UI, `HarnessAgent` wrappers, ~48 providers, 5+ UI frameworks, speech/transcription. Behind Deuz on: zero-dep/dual-format packaging, seam-level determinism, built-in cost tracking, automatic compaction, budget stops, approval-composing sub-agents. Full breakdown: [vs AI SDK 7](/docs/comparison/ai-sdk-7).

### Mastra — Tier 2, docs-level

`@mastra/core@1.46.0` (2026-06-24); 1.0 stable January 2026; Apache-2.0 with source-available `ee/` directories under a commercial enterprise license. A *framework*, not an SDK: graph-based workflow engine with `suspend()`/`resume()` and storage-backed snapshots that survive restarts and deployments; observational memory (compresses old conversations); built-in RAG; a local debugging Studio; a model router advertising 4,000+ models; OTel-compatible observability with an experimental OTel bridge (March 2026).

Trade-off versus Deuz: you adopt Mastra's structure, storage and dependency tree; Deuz gives you primitives you compose yourself with zero runtime dependencies. One relevant 2026 datapoint for that trade-off: in June 2026 the `@mastra/*` npm scope (140+ packages) was briefly republished with a malicious dependency after an organization-credential compromise — cleaned up quickly, but it illustrates the supply-chain surface that a large dependency tree carries. Deuz's core ships zero runtime dependencies precisely to keep that surface minimal.

### LangGraph.js — Tier 2, docs-level

`@langchain/langgraph` (MIT; used in production by Replit, Uber, LinkedIn, GitLab per its README). The strongest durable-execution story in the TypeScript ecosystem today: a checkpointer persists graph state at every super-step, with pluggable backends (in-memory, SQLite, Postgres, Redis, MongoDB) and three durability modes (`exit` / `async` / `sync`); `interrupt()` gives human-in-the-loop that survives restarts; `getStateHistory` enables time-travel debugging.

Worth noting: LangGraph documents the same honest resume boundary Deuz's v1.5 durable runtime ships — on resume, the interrupted unit re-runs from its start (a node there, a step here), so side effects must be idempotent. Trade-off versus Deuz: LangGraph is a runtime with a vocabulary to learn (nodes, edges, reducers, threads) and an ecosystem-sized dependency tree; Deuz keeps the loop a free function and (since v1.5) attaches checkpoints to it via one storage seam.

### OpenAI Agents SDK (JS) — Tier 2, docs-level

`@openai/agents@0.12.1` (MIT; requires `zod` v4; Node ≥22, Deno, Bun). Small, well-designed primitive set: agents, handoffs / agents-as-tools, input+output guardrails, sessions (automatic history), MCP tools, built-in tracing to the OpenAI dashboard, human-in-the-loop via `needsApproval: true` with resumable run state, beta sandbox agents (filesystem workspace), and realtime voice agents. Provider-agnostic in principle (custom `Model` interface), OpenAI-native in practice — tracing and evals live in OpenAI's platform.

Trade-off versus Deuz: the Agents SDK optimizes for minimal code on OpenAI's stack; Deuz optimizes for provider-neutrality, wire ownership and deterministic testing. If your bet is "OpenAI only, forever", their SDK rewards it; if you want to move when the price-performance frontier moves, a canonical multi-provider stream is the safer foundation.
